#  Gove.AI | AI-Copilot для ЧСИ
**«Palantir для исполнительного производства и Налоговой»**

**Хакатон:** Decentrathon 5.0  
**Трек:** AI inDrive (AI for Government)  
**Кейс:** Свободный кейс (B2B сектор / Интеграция с Цифровым тенге / Обеление экономики)


##  Проблема (Гильотина для бизнеса)
В текущей финансовой системе РК арест расчетного счета компании (по линии ЧСИ или КГД) работает как гильотина: счет блокируется на 100%. Бизнес не может выплачивать зарплаты и налоги. Работающее предприятие доводится до банкротства, люди уходят в тень, а государство и кредиторы не получают долг.


##  Решение (Умный Регулятор)
**Gove.AI** - это интеллектуальный AI-Copilot для судебных исполнителей. Мы заменяем тотальную блокировку на **динамическое сплитование финансовых потоков** строго под контролем человека (Human-in-the-Loop). ИИ не заменяет инспектора, а выступает NLP-аналитиком:
* **0% (Обеление ФОТ):** Платежи на зарплаты и налоги помечаются как безопасные.
* **15% (Сплитование):** Для легитимной B2B-выручки ИИ рассчитывает 15% удержания, оставляя бизнесу кислород.
* **100% (Блок Фрода):** ИИ использует NLP для семантического анализа поля «Назначение платежа» и сопоставляет его с ОКЭД получателя. Если строительная компания платит за «IT-консалтинг» учредителю (данные из API КГД), ИИ выдает красный флаг и рекомендует ЧСИ заблокировать транзакцию. Финальное решение всегда за человеком.


##  Источники данных и база знаний ИИ
Логика принятия решений ИИ базируется на кросс-анализе данных из официальных государственных источников и внутренних реестров:

* **Реестр должников (КГД МФ РК):** Использование данных API «Список должников, признанных банкротами или в процедуре реабилитации».
* **Анализ аффилированности:** ИИ сверяет `receiver_iin` (ИИН получателя) с массивами `founderData` (учредители) и `directorData` (руководители) из официальных ответов API.
* **Классификаторы РК:** * **КНП (Коды назначения платежей):** Идентификация защищенных выплат (зарплаты, алименты, налоги).
    * **ОКЭД (Виды деятельности):** Валидация соответствия деятельности компании назначению платежа для выявления фиктивных сделок.
* **Blockchain-оракул:** Фиксация неизменяемого следа транзакции в сети Solana (эмулятор Цифрового тенге) для аудита действий ЧСИ.


##  Архитектура решения (MVP)

Система построена на принципах прозрачности и скорости взаимодействия с государственными реестрами и блокчейном. Особое внимание уделено защите персональных данных (ПДН) и банковской тайны.

* **Frontend (Ситуационный центр ЧСИ):** Vanilla JS + Tailwind CSS. Строгий дашборд в стилистике АИС ОИП / e-Gov для мониторинга рисковых транзакций в реальном времени.
* **Backend (Серверная часть):** Python (FastAPI). Маршрутизация API-запросов, обработка логики удержаний.
* **Database (Хранилище данных):** SQLite. Локальный реестр должников, история транзакций и статусов удержания.
* **AI Core (Интеллектуальное ядро):** Для демонстрации MVP используется API Google Gemini 1.5 Flash. **В Production-версии** осуществляется переход на Sovereign AI — вычислительное ядро на базе локально развернутой Open-Source LLM (например, Llama 3) в закрытом On-Premise контуре (на серверах АО "НИТ" или Нацбанка РК). Это гарантирует 100% сохранность конфиденциальных данных. ИИ выполняет кросс-валидацию КНП и ОКЭД, принимает решение о сплитовании и формирует Explainability-справку.
* **Integration & Execution Layer (Исполнительный слой):** Система спроектирована по двухэтапной модели:
  * **Этап 1 (Go-to-Market):** Интеграция с БВУ через Open API в рамках государственной песочницы (Smart Data Ukimet). Gove.AI выступает как шлюз, отправляя банкам команду на сплитование.
  * **Этап 2 (Future-Proof):** Готовность к инфраструктуре Цифрового тенге (CBDC). В MVP блокчейн Solana Devnet используется как высокоскоростной эмулятор, демонстрируя мгновенное on-chain списание.
* **Alerts (Уведомления):** Telegram Bot API. Автоматическое информирование руководства компании-должника о произведенном списании. Включает официальную выписку, сумму удержания и остаток задолженности, что исключает эффект «внезапной блокировки» и позволяет бизнесу продолжать работу.


##  Техническая документация и запуск (для жюри)

**Запуск проекта локально:**
1. Клонируйте репозиторий: https://github.com/mashurovanargiznm-gif/Gove.AI-MVP.
2. Установите зависимости (Python 3.9+ / Node.js): `pip install -r requirements.txt` и `npm install`
3. Добавьте `.env` файл с ключами (Gemini API, Solana Private Key, Telegram Token).
4. Запустите сервер: `uvicorn main:app --reload`
5. Откройте `index.html` в браузере (или используйте Live Server).

**Сценарий применения (Demo):**
В интерфейсе «Эмулятор банка» введите ИИН получателя `010101010101` (зашит в Mock-API как учредитель-банкрот) и сумму 1 500 000 ₸. Наблюдайте, как ИИ формирует Explainability-справку о фроде. При нажатии «Утвердить» происходит демо-списание 0.2 SOL в сети Devnet.

** Ограничения MVP (Limitations):**
* В связи с отсутствием публичного API Цифрового тенге, исполнительный on-chain слой эмулируется в сети Solana Devnet.
* API КГД МФ РК (реестр должников) на данном этапе симулируется через Mock-сервис, возвращающий JSON в строгом соответствии с официальной документацией ведомства.
* Текущая NLP-модель оптимизирована под русскоязычные назначения платежей, казахский язык поддерживается в режиме беты.
